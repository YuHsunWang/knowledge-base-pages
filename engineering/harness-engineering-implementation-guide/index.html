    Harness Engineering 線束工程實作指南¶

    本文將 harness engineering 解讀為 wire / wiring harness engineering（線束工程）：把電氣連接需求轉成可製造、可安裝、可檢驗、可維修的線束產品與工程資料。

    一句話結論¶

    線束工程的重點不是「把線接起來」，而是建立一套從需求、電氣架構、3D 路由、線材/端子選型、製造圖、BOM、測試到變更控制都能串起來的資料流程。若沒有這套流程，問題通常會出現在量產後：線長不準、端子錯配、干涉、壓接不良、版本混亂、維修困難與成本失控。

    什麼是 Harness Engineering？¶

    線束工程是跨領域工程，介於電氣設計、機構設計、製造工程與品質工程之間。

    它要回答幾個問題：

    • 哪些電源、訊號、通訊線路需要連接？
    • 每條線需要多大電流、電壓、溫度、抗干擾與機械保護？
    • 線要走哪條路徑，如何避開熱源、銳邊、運動件與 EMI 風險？
    • 要用什麼接插件、端子、線材、保護套管、固定件與標籤？
    • 製造現場如何切線、剝線、壓接、組裝、包覆、檢驗？
    • 需求變更後，如何同步圖面、BOM、測試與現場工法？

    核心交付物¶

    一個成熟的線束工程流程，至少應交付以下資料：

    • 電氣連接資料：from-to list、接插件 pin-out、線號、線徑、顏色、訊號名稱。
    • 機構路由資料：3D routing、分支點、固定點、彎曲半徑、與周邊零件間隙。
    • 材料與 BOM：線材、端子、接插件、seal、clip、tube、tape、label、保護套管。
    • 製造資料：formboard / nailboard、切線長度、剝線長度、壓接規格、包覆方式、標籤位置。
    • 品質驗證資料：導通、短路、耐壓、壓接拉力、外觀、尺寸、標籤與追溯碼。
    • 變更紀錄：版本、ECR / ECO、變更原因、影響範圍、舊版處理方式。

    推薦實作流程¶

    1. 需求定義：先建立「線束需求表」¶

    不要一開始就畫線。先把需求整理成表格：

    • 系統 / 子系統
    • 電源、接地、訊號、通訊類型
    • 電流、電壓、功率、保險絲需求
    • 使用環境：溫度、水氣、油污、震動、彎折、戶外 / 車內 / 機箱內
    • 法規或客戶規範
    • 可維修性與裝配順序

    建議：第一版可用 spreadsheet，但每一列都要有唯一 ID。後續所有 schematic、BOM、測試項目都要能追回這個 ID。

    2. 電氣架構：建立 from-to 與保護策略¶

    這一步要把功能需求轉成電氣連接：

    • 定義 ECU / 控制器 / 感測器 / 致動器 / 電源模組。
    • 建立 connector pin-out。
    • 建立 from-to list。
    • 確認電源分配、保險絲、接地點與 shield drain。
    • 區分高電流、低電平訊號、通訊線與高 EMI 風險線路。

    常見錯誤：只做點對點連線，沒有同步定義保護、接地、保險絲與 EMI 策略，後面很容易返工。

    3. 線材、端子與接插件選型¶

    選型不只看電流，還要看環境與製造能力：

    • 線徑：電流、壓降、溫升、束線降額。
    • 絕緣材料：PVC、XLPE、Teflon 等，依溫度、柔軟度與成本選擇。
    • 端子：壓接範圍、鍍層、拉力要求、工具相容性。
    • 接插件：防水等級、lock、CPA/TPA、pin 密度、維修便利性。
    • 保護材料：corrugated tube、braid、heat shrink、tape、grommet。

    建議：建立 approved parts list。不要每個專案重新選料，否則 BOM、採購與品質管理會失控。

    4. 3D Routing：把線束放進真實空間¶

    3D routing 的目標不是漂亮，而是確認能裝、能用、能修：

    • 避開熱源、銳邊與運動件。
    • 保留裝配公差與維修空間。
    • 控制彎曲半徑與拉力。
    • 確認固定點、clip、grommet 與穿孔位置。
    • 確認線束不會和鈑金、機構件、管路干涉。

    建議：電氣工程師與機構工程師要共同 review。線束通常是系統最後才被放進去，但最容易被所有部件擠壓。

    5. 製造工程：從設計轉成 formboard 與工法¶

    量產需要的不只是設計圖，而是製造可執行資料：

    • formboard / nailboard 展開圖
    • 分支點、固定點與標籤位置
    • 切線長度與剝線長度
    • 壓接端子與壓接工具
    • 熱縮、包膠、套管、綁扎規格
    • 製程檢查點

    建議：設計端要及早和製造端 review。若設計圖到現場才發現無法壓接、無法套管、無法裝配，返工成本會很高。

    6. 品質驗證：把標準變成檢查表¶

    線束品質不應只靠終檢，應該分成製程內檢查與出貨檢查：

    • 導通與短路測試
    • 耐壓 / 絕緣測試（依產品需求）
    • 壓接高度、壓接外觀、拉力測試
    • connector pin 位置與鎖扣確認
    • 尺寸、分支長度、標籤與外觀
    • 防水件、seal、grommet 安裝狀態

    IPC/WHMA-A-620 常被用作 cable / wire harness assembly 的驗收與品質語言；航太或高可靠度場景則可參考 NASA 的 cable/harness workmanship 標準思路。

    7. 變更控制：避免圖面、BOM、現場版本不同步¶

    線束最怕「圖改了，但 BOM 沒改」、「BOM 改了，但 formboard 沒改」、「現場照舊版做」。因此需要：

    • 每條線、每個接插件、每個材料都有唯一料號或 ID。
    • 每次變更都記錄原因、影響範圍與生效版本。
    • 設計、採購、製造、測試使用同一套版本。
    • 舊版庫存如何處理要明確。

    工具選擇建議¶

    小型團隊 / 初期專案¶

    適合用：spreadsheet + 2D CAD + 3D CAD + 明確命名規則。

    最低要求：

    • from-to list 有唯一 ID。
    • BOM、線表、測試表能互相對應。
    • 每次改版有版本號與變更紀錄。

    中大型產品 / 多版本產品¶

    建議導入模型化 E/E 系統設計工具，例如 Siemens Capital、Zuken E3.series、CATIA Electrical 等級的解決方案。重點不是品牌，而是要讓 schematic、3D routing、BOM、formboard 與測試資料從同一份資料模型輸出。

    導入前先問三個問題：

    1. 現在最大的痛點是設計錯誤、版本混亂、製造效率，還是報價速度？
    2. 是否已有穩定的零件庫、接插件庫與命名規則？
    3. 工程、製造、採購與品質是否願意共用同一份資料源？

    如果答案是否定的，先導入流程與資料規範，再導入工具；否則工具只會把混亂數位化。

    導入路線圖¶

    第 0 階段：盤點現況¶

    • 蒐集現有線表、BOM、圖面、測試表與變更紀錄。
    • 找出最常出錯的 10 類問題。
    • 統一命名規則：connector、wire、terminal、splice、branch、bundle。

    第 1 階段：建立資料基準¶

    • 建立 approved parts list。
    • 建立標準線表欄位。
    • 建立 connector pin-out 模板。
    • 建立基本設計規則：線徑、顏色、彎曲半徑、固定距離、保護材料。

    第 2 階段：導入設計審查¶

    每次設計 release 前至少檢查：

    • 電流 / 線徑 / 保險絲是否一致。
    • 接插件與端子是否相容。
    • routing 是否干涉或過度彎折。
    • BOM 是否與圖面一致。
    • formboard 是否能製造。
    • 測試表是否涵蓋所有線路。

    第 3 階段：半自動化¶

    • 從 from-to list 自動產生測試清單。
    • 從 BOM 自動檢查未核准料號。
    • 從線表自動檢查缺失欄位。
    • 用版本號把設計、BOM、製造與測試綁定。

    第 4 階段：模型化與 PLM 整合¶

    • schematic、3D routing、formboard、BOM、測試資料共用資料模型。
    • 與 PLM / ERP / MES 串接。
    • 建立變更影響分析。
    • 建立製造現場的 digital work instruction。

    實作檢查表¶

    • [ ] 每條線是否有唯一 ID？
    • [ ] 每個 connector pin 是否有訊號名稱、方向與端點？
    • [ ] 線徑是否經過電流、壓降與溫升檢查？
    • [ ] fuse / breaker 是否能保護線材？
    • [ ] routing 是否避開熱源、銳邊與運動件？
    • [ ] 彎曲半徑與固定距離是否符合規範？
    • [ ] 端子、線材與壓接工具是否相容？
    • [ ] BOM、線表、圖面與測試表是否同版本？
    • [ ] 是否有壓接拉力、導通、短路與外觀檢查？
    • [ ] 變更後是否同步更新 formboard 與現場工法？

    我的建議¶

    如果你要實際導入，我會建議不要一開始追求完整工具平台，而是先做三件事：

    1. 建立標準資料表：需求表、from-to list、BOM、測試表先統一欄位與 ID。
    2. 建立設計規則：線徑、端子、接插件、保護材料、routing、檢驗標準先文件化。
    3. 建立 release gate：每次 release 前用 checklist 檢查電氣、機構、製造、品質與版本一致性。

    等這三件事穩定後，再導入模型化工具。否則導入工具只會加速產生更多不一致資料。
